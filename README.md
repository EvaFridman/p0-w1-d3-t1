# p0-w1-d3-t1 · Что слышит скринридер

> Тип: короткая отработка · Репозиторий: один отдельный

## Контекст

Часть пользователей не видит экран и слушает страницу через скринридер. Если у картинки нет
`alt`, а у кнопки-иконки нет имени — человек слышит «изображение», «кнопка» и не понимает, что
это. Сегодня учишься давать каждому элементу доступное имя.

```
что «видит» скринридер:
┌──────────────────────────────────────────────┐
│ <img>           → читает alt                 │
│ <button>        → читает текст внутри        │
│ иконка без слов → читает aria-label          │
│ поле + подпись  → читает label               │
└──────────────────────────────────────────────┘
нет имени → «кнопка», «изображение» — бесполезно
```

## Что нужно сделать

Взять небольшую страницу (можно семантический каркас из дня 1) и довести её до состояния, когда
каждый смысловой элемент имеет доступное имя: осмысленный `alt` у картинок, текст или
`aria-label` у кнопок-иконок, связь подпись↔поле. Затем проверить ушами — включить любой
скринридер (VoiceOver на Mac: Cmd+F5) и пройтись по странице.

## Релизы

### Release 1 — alt у картинок
Проставить осмысленный `alt` всем `<img>`. Декоративным — пустой `alt=""`. В `A11Y.md` записать
правило: какой alt у смысловой картинки, какой у декоративной.

### Release 2 — имена у интерактивного
Кнопкам с текстом — ничего не нужно. Кнопкам-иконкам (например «закрыть», «меню») — `aria-label`.
Проверить, что у каждой интерактивной штуки есть имя.

### Release 3 — проверка ушами
Включить скринридер, пройти страницу. В `A11Y.md` выписать, что читалось плохо до правок и как
стало после.

## Критерии приёмки

* У каждой смысловой картинки осмысленный `alt`, у декоративных — пустой.
* У каждой кнопки-иконки есть доступное имя (`aria-label` или текст).
* В `A11Y.md` есть заметки по результатам проверки скринридером.
